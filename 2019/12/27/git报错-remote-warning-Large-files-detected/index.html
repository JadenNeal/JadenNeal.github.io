git报错-remote: warning: Large files detected

2019-12-27

本篇是关于git上传大文件报错的问题。

码云的通告：
对于普通用户码云单个仓库限制为1G，单个文件限制100M。如果超过限制,您的代码将无法推送,我们提供的只是代码托管平台, 不是网盘,二进制文件请勿存放在码云,我们会不定期执行仓库大小扫描,如果发现你您的仓库超过大小超过1G,我们会联系您处理, 如果您15个工作日内没有处理,我们将暂停您的仓库的访问,请注意,我们只是暂停您仓库的访问,并不是删除您的仓库, 如果您超过半年没有处理,我们将永久屏蔽该仓库的访问.
如果用户仓库提交超过 1GB,请在本地减小仓库体积后清空远程仓库,重新推送到远程服务器即可.这一策略适用与大文件.
企业用户能够支持更大体积的仓库。

所以，如果一次性上传的文件大小超过100M，则会抛出上述的错误。

那么处理方式如下：

  1. 给这个文件开个后门。
  2. 放弃上传此超标文件。
  3. 修改上传文件大小的限制。

这里只说第一种解决方法。

在bash中输入git filter-branch -f --index-filter "git rm -rf --cached --ignore-unmatch example.zip" -- --all，其中的example.zip就是超标的文件名，当然了，也可以是文件夹，例如:
git filter-branch -f --index-filter "git rm -rf --cached --ignore-unmatch assignment1/cs231n/datasets" -- --all，就上传了datasets文件夹。

上述工作完成后，之前还有一个上传失败的文件夹，在库中显示为文件名，但打不开，于是就想删除它。

这样就删除了不需要的文件。
